### 테마 전환 기능 (컬러 팔레트 스위처)

헤더 우측에 5개 컬러 스와치 칩을 배치하여 클릭 시 사이트 전체 색상이 실시간으로 변경된다.

- `useTheme` 커스텀 훅: CSS 사용자 정의 변수(`--t-from`, `--t-to` 등)를 `document.documentElement`에 동적 적용
- `localStorage`에 선택 테마 저장 → 새로고침 후에도 유지
- `btn-primary`, `tab-btn.active`, 스크롤 버튼 등 주요 UI 요소가 테마 변수를 따름
- 선택된 스와치에 링(ring) 표시로 현재 테마 강조

| 테마 | 그라데이션 | hex |

| 핑크 (기본) | blush → coral | `#EC7A9A` → `#FF6B7A` |
| 블루 | azure → plum | `#3892D8` → `#8B58E8` |

### OG 이미지 리디자인

**변경 내용**
- 기존 다크 네이비 배경 + 영문 타이틀(`AI Personal Shopper`) → 파스텔 핑크 배경 + 한글 타이틀로 전면 교체
- 메인 텍스트: **"AI가 찾아주는 나만의 스타일"**
- 배경: 파스텔 핑크·로즈 그라데이션(`#FFF0F5` → `#FFDDE9`)
- 장식 요소: SVG 장미꽃(우상단 대, 좌하단 소), 파스텔 보케 원, 하단 색상 스와치
- 이미지 생성 스크립트: `scripts/generate-og.cjs` (sharp 기반 SVG → PNG)

**카카오 OG 캐시 문제 및 해결**

카카오 개발자 디버거에서 캐시 초기화 후 재디버그해도 이전 이미지가 계속 표시되는 현상 발생.

- **원인**: 카카오 플랫폼이 이미지 URL 자체를 캐싱하므로 파일 내용이 바뀌어도 같은 URL이면 캐시를 무효화하지 않음
- **해결**: 이미지 파일명을 `og-image.png` → `og-image-v2.png`로 변경하고 `index.html`의 `og:image` / `twitter:image` 메타태그 URL 일괄 수정
- **교훈**: OG 이미지 교체 시 파일명도 함께 변경해야 카카오 등 SNS 플랫폼 캐시를 즉시 무효화할 수 있음

## 2026-06-10 (3차)

### 카카오 로그인 400 오류 수정

**증상**

배포 후 카카오 로그인 버튼 클릭 시 `400 (Bad Request)` 오류 발생.

**원인**

`signInWithOAuth`에 `scopes`를 명시하지 않아 Supabase 대시보드 Kakao Provider 설정값이 기본 scope와 중복 합산됨:
```
scope=account_email+profile_image+profile_nickname+profile_nickname
```
`profile_nickname`이 두 번 포함되어 카카오 서버가 400으로 거부.

**수정**

`src/contexts/AuthContext.jsx` — `signInWithKakao`에 scopes 명시:
```js
scopes: 'profile_nickname profile_image account_email',
```

**확인한 설정**
- Supabase Redirect URL 등록 ✓
- 카카오 콘솔 로그인 Redirect URI 등록 ✓
- 카카오 로그인 활성화 ON ✓
- 이메일 Provider 활성화 ✓
