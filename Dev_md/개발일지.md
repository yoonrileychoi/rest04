### 테마 전환 기능 (컬러 팔레트 스위처)

헤더 우측에 5개 컬러 스와치 칩을 배치하여 클릭 시 사이트 전체 색상이 실시간으로 변경된다.

- `useTheme` 커스텀 훅: CSS 사용자 정의 변수(`--t-from`, `--t-to` 등)를 `document.documentElement`에 동적 적용
- `localStorage`에 선택 테마 저장 → 새로고침 후에도 유지
- `btn-primary`, `tab-btn.active`, 스크롤 버튼 등 주요 UI 요소가 테마 변수를 따름
- 선택된 스와치에 링(ring) 표시로 현재 테마 강조

| 테마 | 그라데이션 | hex |
|------|-----------|-----|
| 핑크 (기본) | blush → coral | `#EC7A9A` → `#FF6B7A` |
| 블루 | azure → plum | `#3892D8` → `#8B58E8` |
| 민트 | sage → azure | `#1AA27F` → `#3892D8` |
| 퍼플 | plum → blush | `#8B58E8` → `#EC7A9A` |
| 코럴 | coral → blush | `#F5465A` → `#EC7A9A` |

### OG 이미지 리디자인

- 기존 다크 네이비 배경 + 영문 타이틀 → 파스텔 핑크 배경 + 한글 타이틀로 전면 교체

- 장식 요소: SVG 장미꽃(우상단 대, 좌하단 소), 파스텔 보케 원, 하단 색상 스와치

카카오 개발자 디버거에서 캐시 초기화 후 재디버그해도 이전 이미지가 계속 표시되는 현상 발생.

- **해결**: 이미지 파일명을 `og-image.png` → `og-image-v2.png`로 변경하고 `index.html`의 `og:image` / `twitter:image` 메타태그 URL 일괄 수정

## 2026-06-11

### 카카오 로그인 · 이메일 회원가입 전면 해결

배포 후 카카오 로그인 및 이메일 회원가입에서 복수의 문제가 발생하여 전면 점검 및 수정하였다.

---

#### 문제 1 — HashRouter와 Supabase OAuth 충돌

HashRouter가 Supabase OAuth 콜백 URL의 `#error=...` 해시를 라우팅 경로로 해석해 페이지가 깨지는 문제 발생.

**수정**
- `src/main.jsx`: `HashRouter` → `BrowserRouter basename="/rest04"`
- `vite.config.js`: 빌드 시 `dist/404.html` 자동 생성 플러그인 추가

---

#### 문제 2 — 카카오 OAuth 동의항목 미등록 (KOE205)

카카오 개발자 콘솔에 일부 동의항목이 등록되지 않아 `KOE205` 오류 발생.

**수정**: 카카오 개발자 콘솔 → 닉네임, 프로필 사진, 카카오계정 이메일 세 항목 모두 등록

---

#### 문제 3 — profiles 테이블 INSERT 정책 누락

`profiles` 테이블에 RLS INSERT 정책이 없어 신규 사용자 프로필 생성 불가.

**수정**
- Supabase SQL Editor에서 INSERT 정책 추가:
```sql
CREATE POLICY "profiles_insert" ON profiles FOR INSERT WITH CHECK (auth.uid() = id);
```
- `handle_new_user` 트리거 제거 → 프로필 생성 로직을 앱 코드로 이관
- `src/contexts/AuthContext.jsx` — 프로필 미존재 시 자동 생성:
```js
if (error?.code === 'PGRST116') {
  const nickname = meta.nickname || meta.full_name || meta.name || '사용자'
  await supabase.from('profiles').insert({ id: userId, nickname })
}
```

---

#### 문제 4 — 카카오 사용자 작성자 이름 오류

`BoardWrite.jsx`에서 `user_metadata.nickname`만 참조해 카카오 유저의 이메일 앞부분이 작성자명으로 표시됨.

**수정**: `src/pages/BoardWrite.jsx` — 작성자명 참조 순서 확장
```js
const nickname =
  user.user_metadata?.nickname ||
  user.user_metadata?.full_name ||
  user.user_metadata?.name ||
  user.email?.split('@')[0] || '익명'
```

---

#### 문제 5 — 이메일 회원가입 후 인증 메시지 표시

회원가입 성공 시 인증 메일 안내 문구가 하드코딩으로 표시됨.

**수정**: `src/pages/Login.jsx` — 회원가입 성공 시 홈(`/`)으로 바로 이동
